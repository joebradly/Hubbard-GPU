1 Fermi-Hubbard on the GPU with Cuda
2 ----------------------------------
3 
4 This program can calculate the groundstate energy of 1 dimensional and
5 2 dimension Fermi-Hubbard model on the GPU (and the CPU). It is based
6 on [Exact diagonalization of the Hubbard model on graphics processing units](http://arxiv.org/abs/1204.3425)
7 by Siro and Harju.
8 
9 There are 2 programs: main and main2D. The former is for 1D Hubbard,
10 the latter is for 2D Hubbard. Everything is split up in several classes
11 to make reusing code easy: The Hamiltonian and HubHam2D classes build the
12 full (dense) Hamiltonian Matrix. SparseHamiltonian stores the Hubbard
13 Hamiltonian in parts: an spin up and a spin down part. The matrix
14 themselves are storred in the ELL format. The SparseHamiltonian2D does
15 the same but for 2D Hubbard. However, here we make a detour: we first
16 store the matrices in the CRS format (the SparseHamiltonian2DCSR class)
17 and then convert it in the ELL format. The reason is that for ELL,
18 we need to know the maximum number of non-zero elements (nnz) of a row.
19 
20 There are several branches in the git repo: the master contains
21 only the CPU version. The branch 'GPU' constains the GPU version
22 and the branch 'PRIMME' used the PRIMME library to find the
23 eigenvalues and eigenvectors. You can find PRIMME
24 at [http://www.cs.wm.edu/~andreas/software/](http://www.cs.wm.edu/~andreas/software/)
